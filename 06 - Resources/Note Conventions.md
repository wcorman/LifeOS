# Note Conventions

## Frontmatter
Use simple frontmatter at the top of notes when helpful:

```yaml
---
type: project | daily-note | knowledge | learning | life-area | resource
status: active | paused | archived
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - example
---
```

## Linking Style
- Prefer `[[wikilinks]]` for internal notes
- Link across folders when ideas genuinely connect
- Avoid creating links just for the sake of density

## Naming
- Use clear note titles
- Keep folder names stable
- Let structure emerge instead of over-optimizing early

## Rule of Thumb
Create notes that are easy to revisit, not just satisfying to organize.
